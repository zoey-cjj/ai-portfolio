📘 Case 1：族谱智能问答助手（RAG）

1️⃣ 项目背景

族谱数据多为非结构化文本，用户难以通过关键词检索复杂亲缘关系。

2️⃣ 产品目标

  • 自然语言提问，降低使用门槛
  • 回答可追溯，避免模型幻觉
  • 支持多代、跨支系关系推理

3️⃣ 方案设计

文档解析 → 分块 → Embedding → 向量检索 → LLM 生成 → 来源标注

4️⃣ 评测体系

  • 准确性 / 完整性 / 幻觉 / 合规
  • Golden Set：20 条典型问题

📘 Case 2：中东支付风控助手

1️⃣ 业务背景

中东地区支付监管复杂，拒付与欺诈风险高，人工审核效率低。

2️⃣ 产品目标

  • 降低人工审核成本
  • 提升高风险交易识别准确率
  • 减少误杀正常交易

3️⃣ AI 解法

使用 LLM 解读风控规则与交易上下文，辅助审核人员决策。

4️⃣ 产品价值

  • 审核效率显著提升
  • 风控规则可解释性强
  • 支持多语言交易场景

🧪 评测体系（Eval）

分数 标准
5 完全正确，无幻觉
3 部分正确
1 错误 / 有害
