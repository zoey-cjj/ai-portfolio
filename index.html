AI Case – 族谱知识问答助手 (RAG MVP)

背景：族谱产品中包含大量人物关系、辈分、世系文本，用户难以通过传统关键词搜索快速定位复杂亲缘关系。

方案：构建一个基于 RAG (检索增强生成) 的应用。将族谱文本向量化存入知识库，用户提问时，系统检索最相关的片段，交由 LLM 生成解释性回答并注明出处。

产品说明

  • 知识库构建：将 PDF/Word 族谱文档切分为 300-500 Token 的片段，存入向量数据库。
  • 检索策略：使用 Embedding 模型将用户问题向量化，在库中召回 Top-K 相关片段。
  • 生成约束：通过 Prompt 工程，硬性要求模型仅基于检索到的片段作答，无法回答时需声明。
  • MVP 范围：单轮问答、封闭域知识库、简单事实型查询。

Prompt Lab – 族谱问答 Prompt 设计

通过 System Prompt 约束模型行为，降低幻觉；User Prompt 明确上下文与输出格式。

System Prompt

你是一个专业的族谱知识问答助手。
请严格遵守以下规则：
1. 只允许根据【知识库】内容回答，绝对禁止编造。
2. 如知识库无法回答，必须回复：“当前族谱未收录该信息。”
3. 回答需简洁、专业，尽量注明出处（如“据《某氏家谱》第X卷”）。
4. 若用户问题模糊，可礼貌请求澄清。

User Prompt 模板

【知识库】
{retrieved_passages}

【用户问题】
{user_question}

【回答要求】

- 再给依据
- 最后说明信息是否确定
