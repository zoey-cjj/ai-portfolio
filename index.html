Case 1｜族谱知识问答助手（主 Case）

背景

族谱数据多为非结构化文本，用户难以通过关键词检索复杂亲缘关系。

产品目标

  • 自然语言提问，降低使用门槛
  • 回答可追溯，避免模型幻觉
  • 支持多代、跨支系关系推理

RAG 架构

文档解析 → 分块 → Embedding → 向量检索 → LLM 生成 → 来源标注

Prompt 设计

System Prompt 明确角色与约束；User Prompt 注入检索片段与用户问题。

产品边界

当前支持事实型查询，不支持主观推测与编造。

Case 2｜中东支付风控助手（辅 Case）

业务背景

中东地区支付监管复杂，拒付与欺诈风险高，人工审核效率低。

产品目标

  • 降低人工审核成本
  • 提升高风险交易识别准确率
  • 减少误杀正常交易

AI 解法

使用 LLM 解读风控规则与交易上下文，辅助审核人员决策。

产品价值

  • 审核效率显著提升
  • 风控规则可解释性强
  • 支持多语言交易场景
